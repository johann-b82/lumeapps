# User Management

## Roles

The KPI Dashboard has two roles:

| Role          | Access                                                         |
|---------------|----------------------------------------------------------------|
| Administrator | Full access -- upload data, configure settings, view admin guide |
| Viewer        | Read-only dashboards and user guide only                       |

The Viewer role has a fixed UUID: `a2222222-bbbb-bbbb-bbbb-bbbbbbbbbbbb`, set in the `bootstrap-roles.sh` script during initial deployment.

## Creating a User

1. Open the Directus admin panel at `http://localhost:8055`.
2. Log in with the credentials from your `.env` file (`DIRECTUS_ADMIN_EMAIL` / `DIRECTUS_ADMIN_PASSWORD`).
3. Navigate to **Users** in the left sidebar.
4. Click **+ New User**.
5. Set the user's **email**, **password**, and **role** (Administrator or Viewer).
6. Click **Save**.

The new user can now log in to the KPI Dashboard with the assigned role.

## Promoting a User

1. Open the Directus admin panel at `http://localhost:8055`.
2. Navigate to **Users** in the left sidebar and select the user.
3. Change the **Role** field to **Administrator**.
4. Click **Save**.

> **Tip:** Role changes take effect on the user's next login. If the user is currently logged in, they will need to log out and log back in to see their new permissions.

## Administrator Role UUID

The Administrator role UUID is generated by Directus on first boot and is not fixed like the Viewer role. You must fetch it and set it as `DIRECTUS_ADMINISTRATOR_ROLE_UUID` in your `.env` file for the application to correctly identify administrators.

Refer to the [System Setup](/docs/admin-guide/system-setup) article for instructions on fetching the Administrator role UUID during initial deployment.
